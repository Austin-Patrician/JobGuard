这份**《职场防坑模拟器》产品设计方案 (V1.0)** 将以“剧情闯关”为核心，辅以“实用工具箱”，完全摒弃复杂的传统游戏引擎，依托 Web 端实现高互动性、低门槛和极强的社交传播力。

以下是完整的模块化功能设计：

# 🎮 《职场防坑模拟器》核心产品设计方案

## 一、 产品定位与核心机制

* **产品形态：** 响应式 Web 应用（优先适配移动端竖屏浏览，兼容 PC）。
* **核心受众：** 应届毕业生、转行求职者、缺乏社会经验的年轻职场人。
* **核心循环：** `游戏化体验（降低防备/寓教于乐）` ➡️ `实用工具箱（解决实际问题）` ➡️ `一键生成海报（社交裂变传播）`。
* **全局数值系统：**
* **❤️ 职场生命值 (HP)：** 初始 100。轻信 HR 话术或签下问题合同会扣血，归零即达成“被割韭菜”结局。
* **🛡️ 避坑觉悟值 (EXP)：** 成功识破骗局、找出合同漏洞可累加。用于通关结算评级。



---

## 二、 核心模块一：剧情闯关模式 (The Survival Game)

本模块采用“视觉小说+互动解谜”的形式，配合屏幕震动、打字机特效营造沉浸感。

### 🚩 第一关：黄金眼（虚假 JD 识别）

* **场景设定：** 模拟常见的招聘软件界面（如 BOSS 直聘、智联招聘）。
* **功能交互：**
1. 系统随机派发 3 份职位描述（JD），其中混杂着“招转培”、“跨境电商代运营骗局”等。
2. **找茬玩法：** 用户需要在规定时间内，点击 JD 中觉得“有猫腻”的词句（如“无责底薪8000+不限学历”、“入职前需参加为期一周的带薪实训”）。
3. **AI 导师结算：** 提交后，AI 导师弹出，将用户漏掉的“坑”高亮标红，并以大白话解释背后的套路逻辑。识破越多，觉悟值加得越高。



### 💬 第二关：唇枪舌战（高压面试模拟）

* **场景设定：** 模拟微信聊天界面对话框。
* **功能交互：**
1. **AI 动态生成话术：** 对面的“黑心 HR（由大模型扮演）”会根据用户的回复，抛出诱导性问题。例如：“公司为了防止员工离职，需要通过第三方软件垫付 3000 元培训费，每个月随工资返还，你能接受吗？”
2. **分支选择与自定义回复：** 用户可以选择系统提供的 3 个回复选项，也可以**自行输入文字**对线。
3. **即时判定（震动反馈）：** 如果用户选择妥协，屏幕闪烁红光并伴随“震动”动效，HP 大幅扣除；如果用户强硬反问（如“这符合劳动法吗？”），HR 可能会气急败坏，玩家成功防御。



### 📄 第三关：合同迷宫（不平等条约排雷）

* **场景设定：** 办公桌视角，面前摆着一份密密麻麻的《员工入职及培训协议》。
* **功能交互：**
1. **高亮扫描：** 用户在长文本中寻找陷阱条款。
2. **AI 放大镜（求助技能）：** 玩家若找不到，可点击“AI 放大镜”，系统模拟扫描特效，自动框选出“自愿放弃社保”、“天价违约金”、“隐性贷款分期”等条款，并附上《劳动法》相关条文解释。

## 三、 核心模块二：实用防御工具箱 (The Toolkit)

游戏通关后，引导用户在真实求职场景中使用以下 AI 辅助功能：

### 🛠️ 工具 1：JD/聊天记录“照妖镜”

* **输入方式：** 支持文本粘贴、直接上传手机截图（接入 OCR 识别）。
* **输出报告：**
* **红绿灯评级：** 🟢 安全、🟡 存疑（需警惕）、🔴 高危（强烈建议拉黑）。
* **套路拆解：** 将对方的话术“翻译”成真实意图（例如：对方说“扁平化管理”，AI 提示可能意味着“没有晋升空间且一人身兼数职”）。



### ⚖️ 工具 2：合同极速避雷针

* **输入方式：** 上传 PDF 或拍摄的多张纸质合同照片。
* **功能点：** AI 提取关键信息，生成“风险摘要表”。清晰列出薪资构成是否模糊、是否存在霸王条款、离职限制是否合法等核心信息。

## 四、 核心模块三：避坑情报局 (Community Intel)

这是一个基于 AI 自动治理的轻量级“黑名单与经验共享”社区。

* **热力地图展示：** 首页展示一张全国地图或行业标签云，高亮显示近期“招转培”或“套壳公司”高发的地区和岗位（如：成都-短视频剪辑、广州-跨境运营）。
* **匿名吐槽信箱：** 用户提交自己被骗的经历或疑似骗子公司名称。
* **AI 净网与归纳：** 为防止恶意诽谤，AI 在后台自动脱敏（如隐去具体人名），提取骗术核心逻辑，并自动打上标签（`#贷款陷阱`、`#培训坑`、`#传销变种`）后发布至信息流。

## 五、 视觉动效与游戏感需求 (前端表现层)

为了在不用传统游戏引擎的前提下实现“高级感”，需在前端代码中落实以下表现：

1. **打击感反馈 (Impacts)：** * 掉血时：屏幕级 CSS `Shake`（震动）特效 + 边缘红色渐变闪烁。
* 识破时：类似《逆转裁判》的“异议（OBJECTION）”大字以极高的弹簧动画（Spring Animation）砸向屏幕中央。


2. **叙事感表现 (Narrative)：** * 对话文字必须采用 **打字机音效 + 逐字显现**（Typewriter effect），不要一次性全刷出来，以增加阅读压迫感。
3. **UI 质感：** * 采用“赛博办公风”或“极简拟物风”。合同要有纸张的纹理，AI 分析时要有类似黑客破解的“代码瀑布”或“扫描线”扫过屏幕的动画。

## 六、 社交裂变与留存机制 (Growth)

* **结案海报生成：** 玩家 HP 归零（被骗）或通关后，生成个性化战报海报。
* *文案示例：* “我在职场防坑模拟器中活了 2 关，最终被 HR 的‘画大饼’无情反杀，超越了全国 15% 的应届生。你能在黑心 HR 手里撑多久？”


* **二维码引流：** 海报带有网站二维码，方便用户分享到朋友圈、小红书、大学班级群，实现零成本自然增长。

## 七、 技术栈选型与架构细节 (Technical Architecture)

### 1. 前端表现层 (Frontend & Animation)

前端是决定“游戏感”和“沉浸感”的核心，我们需要在不使用传统游戏引擎的前提下，把 DOM 元素的动效榨干。

* **核心框架：Next.js (React)**
* **细节：** 采用 App Router 模式。利用服务端渲染（SSR）保证极快的首屏加载速度，并实现完美的 SEO（让百度/谷歌能收录你的“避坑指南”和“黑名单公司”）。


* **UI 与样式：Tailwind CSS + Shadcn UI**
* **细节：** 用于快速构建响应式布局、毛玻璃面板、赛博风边框等基础 UI，保证在手机微信端和 PC 端都有原生 App 的体验。


* **微动效与反馈：Framer Motion**
* **细节：** 负责游戏的核心“打击感”。实现卡片的物理惯性拖拽、选错时的屏幕剧烈 `Shake`（震动）、红色警告边缘闪烁、以及对话框平滑弹出。


* **复杂时序与运镜：GSAP (GreenSock)**
* **细节：** 用于实现类似电影的“镜头缩放”。比如在“合同审核”关卡中，控制画面从全局平滑放大到某一行霸王条款的特写。


* **矢量动画点缀：Lottie-Web**
* **细节：** 承载开源的 2D 动画资产。比如“加载中的雷达扫描”、“通关成功的撒花”、“警报灯旋转”等，直接引入 JSON 文件，极小且高清。



### 2. 游戏逻辑与状态管理层 (Game Logic Engine)

我们需要一个纯数据驱动的逻辑引擎，把 UI 展示和游戏剧本完全解耦。

* **状态机：XState 或 Zustand**
* **细节：** * 用 Zustand 管理全局数值（如 HP 生命值、EXP 觉悟值、当前关卡进度）。
* 用 XState 管理复杂的剧本分支逻辑（比如：节点 A 选了选项 1 -> 跳转节点 B 且 HP-20 -> 触发特定 AI Prompt）。




* **剧本引擎 (Script Engine)**
* **细节：** 将每一关的剧情、NPC 设定、对话分支全部 JSON 化。前端代码只负责“解析 JSON 并渲染对应组件”，后续更新关卡只需修改云端的 JSON 文件，无需重新发版代码。



### 3. AI 大脑与数据处理层 (AI & Data Pipeline)

这是防坑模拟器的核心竞争力，需要兼顾中文理解能力和响应速度。

* **大语言模型 (LLM)：DeepSeek-V3 / 阿里通义千问 / OpenAI (GPT-4o)**
* **细节：** 推荐首选国产头部模型（如 DeepSeek 或通义千问），它们对中国特有的职场黑话（如“招转培”、“大小周”、“自愿放弃社保”、“管培生陷阱”）的语境理解极其精准，且 API 成本极低。


* **AI 接口中间件：Vercel AI SDK**
* **细节：** 完美抹平不同 AI 大模型的接口差异，原生支持 **流式输出 (Streaming)**。配合前端，轻松实现类似打字机的 `哒哒哒` 逐字显示效果，减少用户等待时间。


* **多模态识别 (OCR)：百度 AI 开放平台 OCR / 腾讯云 OCR**
* **细节：** 针对“合同极速避雷针”功能，大模型自带的视觉能力有时对密密麻麻的中文合同小字识别率有限。建议先通过专业 OCR 引擎提取精准文本，再喂给大模型进行“风险点逻辑分析”。


* **知识库与 RAG (检索增强生成)：Pinecone 或 Supabase pgvector**
* **细节：** 将《中华人民共和国劳动法》、各省市劳动仲裁典型案例转化为向量数据存储。当 AI 分析合同或 HR 话术时，先从向量库中检索出对应的法律条文，让 AI 的诊断结果“有法可依”，拒绝胡编乱造。



### 4. 后端与数据持久层 (Backend & Storage)

为了追求极速上线和低运维成本，全面拥抱 Serverless 和 BaaS（后端即服务）。

* **云开发平台：基于 PostgreSQL**
* **Database (数据库)：** 存储用户的通关记录、AI 提取后的脱敏黑名单公司数据。
* **Storage (对象存储)：** 临时存储用户上传的合同截图、聊天记录截图（需设置定时清理机制以保护隐私）。




* **API 路由：Next.js Route Handlers**
* **细节：** 无需额外部署后端服务器，直接在 Next.js 内部编写 Node.js 接口，处理与 Supabase、AI 大模型、OCR 服务的安全通信，保护 API Key 不被泄露。
